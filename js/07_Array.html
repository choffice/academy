배열(Array)

같은 이름으로 여러 개의 원소를 지정

  • 배열은 원소에 접근할 때 [index]로 접근
  • 배열은 자체적으로 길이값을 제공(.length)

배열의 선언과 접근

  1. 배열을 쓰는 이유
  2. 개별선언?! const scoreCy = 100;const scoreMs = 100;const scoreJW = 100;//번거로움
  3. 배열선언?! let scores = [99,74,82,65,100]//하나의 이름으로 여러 개 값
  4. 접근1?! 코드보기
  5. 접근2?! 코드보기

배열의 함수

  1. 탐색할 땐 indexOf
  2. indexOf?! console.log(let s1=[1,2,3];console.log(s1.indexOf(2)););!!//index는 0부터 시작
  3. 없다면?! console.log(let s1=[1,2,3];console.log(s1.indexOf(7)););!!//배열에 없는 원소 탐색시 -1 반환

  1. 추가할 땐 push, unshift / 제거할 땐 pop, shift
  2. push?! console.log(let s1=[1,2,3];s1.push(6);console.log(s1););!!//맨 마지막에 추가
  3. unshift?! console.log(let s1=[1,2,3];s1.unshift(9);console.log(s1););!!//맨 얖에 추가
  4. pop?! console.log(let s1=[1,2,3];console.log(s1.pop());console.log(s1););!!//마지막 원소 제거
  5. shift?! console.log(let s1=[1,2,3];console.log(s1.shift());console.log(s1););!!//첫 번째 원소 제거

  1. splice의 용도
  2. 1개 혹은 2개의 인수는 삭제
  3. 첫 번째에 위치, 두 번째 인수에 0을 쓰면 3번째 인수부터 추가
  4. 자르기(1)?! console.log(let s1=[1,2,3];console.log(s1.splice(0,1));console.log(s1););!!//원소 추출 및 원본 갱신(원본 배열에 영향o)
  5. 자르기(2)?! console.log(let s1=[1,2,3,5,7];console.log(s1.splice(3));console.log(s1););!!//인수 하나면 우측에서 해당 인덱스까지 자름
  6. 교체하기?! console.log(let s1=[1,2,3];console.log(s1.splice(0,1,8));console.log(s1););!!//0번째에 1개만큼을 8로 교체(인수2는 삭제할 길이)
  7. 더하기(1)?! console.log(let s1=[1,2,3];console.log(s1.splice(0,0,5));console.log(s1););!!//0,0으로 시작하면 맨 앞에 더해짐
  8. 더하기(2)?! console.log(let s1=[1,2,3];console.log(s1.splice(3,0,5));console.log(s1););!!//배열의 길이로 시작하면 맨 끝에 더해짐
  9. 더하기(3)?! console.log(let s1=[1,2,3];console.log(s1.splice(2,0,5));console.log(s1););!!//몇 번째 인덱스에 추가할 건지
