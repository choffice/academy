논리 연산자

true를 1, false를 0, ||를 +, &&를 *으로 치환하여 0보다 큰지 본다


입력 || 논리합 && 논리곱
true true true true
true false true false
false true true false
false false false false
  • Truthy value(true) 1) 0을 제외한 정수, 실수, Infinity, -Infinity
  • Truthy value(true) 2) 빈문자열("")을 제외한 모든 문자열
  • Truthy value(true) 3) 참조형 객체(Object, 내장객체, 배열, class)
  • Truthy value(true) 4) {} 빈 객체, [] 빈 배열, function
  • Truthy value(true) 5) DOM, BOM 사전 정의된 객체들
  • Falsy value(false) 1) undefined, null
  • Falsy value(false) 2) NaN (Not a Number)
  • Falsy value(false) 3) 0, -0
  • Falsy value(false) 4) "", '', 빈문자열
  • Falsy value(false) 5) document.all, alert() 반환되는 값이 없어서 undefined가 됨

연산자의 우선순위

  • 비교 연산자를 논리 연산자보다 우선한다
  • &&논리곱이 ||논리합에 우선한다
  • &&는 왼쪽이 falsy면 그대로 반환, truthy면 오른쪽 값을 반환한다
  • ||는 왼쪽이 truthy면 그대로 반환, falsy면 오른쪽 값을 반환한다

  1. 논리곱의 예// const b1 = 1 < 2; const b2=false;
  2. console.log(b1 && b2); //true false의 논리곱 == false (1*0==0)
  3. console.log(b1 && !b2); //true true의 논리곱 == true (1*1==1)
  4. console.log(!b1 && (!b2 || 'A') < 'a' ); // 괄호 안 먼저, (1+1)=true(1)
  5. console.log('1:', !b2 || 'A'); //괄호 안 먼저, (1+1)=true(1)
  6. console.log('2:', true < 'a' ); //비교연산자 먼저 1 < 1==false(0)
  7. console.log('3:', !b1 && false); // 0*0 = 0(false)
  1. 논리곱의 나열에서 살아남는 값
  2. console.log(1 && -1 && undefined); //t&t&f 인데 왜 마지막 값이지...?
  3. console.log(0 && null && undefined); //첫 번째가 false이므로 그대로 0
  4. console.log([] && {} && 1); //t && t && t 이므로 마지막 1
  5. console.log('' && {} && 1); //첫 번째가 빈문자열이므로 그대로 빈값

단항 연산자

++ 과 --

  • 출력 시점이 중요
  • n++을 그대로 출력하면 실행 전의 n이 출력
  • n++을 실행 후 출력하면 n+1이 출력
  • ++n을 그대로 출력하면 n+1이 출력
  • ++n은 현재의 n+1 그 자체다, ++n은 값이고 n++는 과정이다

shift 연산자

( >> , << )

  • 101 이 왜 5일까?
  • 십진수 5는 이진수에서 0+4+0+1로 계산(오른쪽부터 2의 0~n제곱)
  • 5<<1 은 왜 10일까?
  • <<n는 모든 비트를 왼쪽으로 n칸 이동, 즉 5에서 1칸 이동시 오른쪽에 0이 채워지고 1010이 된다. 1010은 8+0+2+0이므로 10
    1. 예제
    2. console.log(5<<1); //10
    3. console.log(10>>1);//5
